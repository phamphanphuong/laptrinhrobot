Khóa Học: HTML, CSS & JS Thực Chiến

Mục tiêu khóa học: Loại bỏ các khái niệm lý thuyết hàn lâm để tập trung "thực chiến". Bạn sẽ thấy việc lập trình web thực ra rất logic, thú vị và có thể tạo ra sản phẩm mang tên mình ngay từ buổi đầu tiên. Kết thúc 10 buổi, bạn sẽ tự tay hoàn thiện một trang Portfolio cá nhân, ứng dụng tính BMI và đưa lên mạng Internet toàn cầu.

Buổi 1 NHẬP MÔN - "HACK" GIAO DIỆN (90 Phút)

Phần 1: Khởi động & Mindset (15 Phút)

1. Lời mở đầu:

"Chào bạn. Có thể bạn đam mê công nghệ, thích thể thao, mê game, nhưng chưa từng viết code. Không sao cả! Hôm nay chúng ta sẽ không học lý thuyết khô khan. Mục tiêu của buổi hôm nay là sau 90 phút, bạn sẽ có một trang hồ sơ cá nhân mang phong cách cực ngầu, chạy ngay trên máy tính của mình."

2. Hiểu bản chất Web qua góc nhìn Game thủ/Kỹ sư:

Việc xây dựng một trang web giống hệt như việc bạn tự build một dàn PC:

  • HTML: Chính là bộ khung xương, là Mainboard, CPU, vỏ case. Nó tạo ra cấu trúc nền tảng.
  • CSS: Là thiết kế bên ngoài, là đèn LED RGB, tản nhiệt nước. CSS giúp web có màu sắc, bố cục và hiệu ứng đẹp mắt.
  • JavaScript (JS): Là hệ điều hành, là luồng xử lý. JS giúp web 'sống' và có thể tương tác (bấm nút tính toán, chuyển động...).

3. Setup công cụ làm việc:

  • Phần mềm code: Tải và cài đặt trình soạn thảo Visual Studio Code (VS Code).
  • Cài Extension: Tìm và cài đặt tiện ích Live Server (Tiện ích này giúp web tự động cập nhật ngay khi bạn vừa gõ code xong mà không cần tải lại trang).

Phần 2: Viết những dòng code đầu tiên (45 Phút)

1. Tạo thư mục làm việc:

Tạo một thư mục tên Buoi_1_Web ngoài Desktop. Kéo thả thư mục đó vào giao diện VS Code.

Trong VS Code, tạo một file tên là index.html. (File mang tên index.html luôn được các máy chủ ngầm hiểu là trang chủ của mọi website).

2. Khởi tạo cấu trúc xương (HTML Boilerplate):

Mẹo: Gõ dấu ! rồi ấn Enter. VS Code sẽ tự sinh ra bộ khung chuẩn của mọi trang web.

Giải thích chi tiết các thẻ:

<!DOCTYPE html> <!-- Báo cho trình duyệt biết đây là file HTML5 mới nhất -->
<html lang="en"> <!-- Ngôn ngữ web là tiếng Anh -->
<head>
    <meta charset="UTF-8"> <!-- Giúp hiển thị tiếng Việt có dấu, không bị lỗi font -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- Hỗ trợ xem tốt trên điện thoại -->
    <title>Document</title> <!-- Tiêu đề hiển thị trên tab trình duyệt -->
</head>
<body>
    <!-- MỌI THỨ NGƯỜI DÙNG NHÌN THẤY SẼ NẰM TRONG KHU VỰC THẺ BODY NÀY -->
</body>
</html>

3. Đưa nội dung vào <body>:

Thực hành: Sửa lại phần <title>Document</title> thành <title>Hồ sơ của [Tên bạn]</title>. Sau đó click chuột phải vào màn hình code, chọn Open with Live Server.

Gõ đoạn mã sau vào giữa thẻ <body> và </body>:

<div class="card">
    <img src="https://i.pinimg.com/736x/88/5c/15/885c15f48cd114ea0f69f212f845d050.jpg" alt="Avatar">
    <h1>Xin chào, mình là [Tên Của Bạn]</h1>
    <p>Học sinh trường XYZ | Đam mê công nghệ & Thể thao</p>
    <div>
        <span class="tag">🎮 Gamer</span>
        <span class="tag">🏸 Cầu lông</span>
        <span class="tag">🏋️‍♂️ Gym</span>
    </div>
</div>

💡 Từ vựng HTML:

  • <div>: Cái hộp chứa đồ (Box). Dùng để gom nhóm các phần tử.
  • <img>: Chèn hình ảnh. Sử dụng thuộc tính src="" để dán link ảnh.
  • <h1>: Heading 1 - Tiêu đề lớn nhất trang.
  • <p>: Paragraph - Đoạn văn bản bình thường.
  • <span>: Thẻ nhãn dùng cho các khối văn bản nhỏ, nằm trên cùng một hàng.

Phần 3: Mặc áo cho Web bằng CSS (30 Phút)

Phân tích: Trang web hiện tại chỉ có chữ đen nền trắng. Chúng ta sẽ dùng CSS để tạo phong cách giao diện Dark Mode xịn xò.

Thực hành: Chèn đoạn mã CSS sau vào trong thẻ <head>, ngay dưới <title>. Lưu ý gõ cẩn thận từng ngoặc nhọn { }.

<style>
body {
    background-color: #121212; /* Nền màu đen nhám */
    color: #ffffff; /* Chữ màu trắng */
    font-family: Arial, sans-serif; /* Đổi font chữ cho hiện đại */
    text-align: center; /* Căn giữa mọi thứ */
    padding: 50px; /* Cách mép màn hình 50px */
}

.card {
    background-color: #1e1e1e; /* Hộp màu xám đen */
    padding: 30px;
    border-radius: 15px; /* Bo góc hộp cho tròn trịa giống app iPhone */
    box-shadow: 0 4px 15px rgba(0,255,128,0.2); /* Đổ bóng màu xanh lá cây */
    max-width: 400px;
    margin: auto; /* Đẩy cái hộp ra giữa màn hình */
}

img {
    width: 150px;
    height: 150px;
    border-radius: 50%; /* Cắt hình thành hình tròn */
    border: 3px solid #00ff80; /* Viền màu xanh lá */
    object-fit: cover;
}

h1 { color: #00ff80; } /* Tên sẽ sáng màu xanh */

.tag {
    background: #333;
    padding: 5px 15px;
    border-radius: 20px;
    display: inline-block;
    margin: 5px;
}
</style>

🔥 Tổng kết buổi 1:

Chỉ với vài dòng lệnh cơ bản, giao diện đã thay đổi hoàn toàn. Xuyên suốt lộ trình này, bạn sẽ tự tay thiết kế giao diện từ trang giấy trắng, tạo ra bộ công cụ tính toán riêng, và đưa website này lên mạng Internet bằng tên miền để gửi cho bạn bè.

Buổi 2 XÂY DỰNG NỀN MÓNG HTML & LIÊN KẾT ĐA TRANG (90 Phút)

Phần 1: Ôn tập & Setup Dự án Portfolio (20 Phút)

Giới thiệu: Buổi trước chúng ta đã "thử sức" với code. Hôm nay, chúng ta sẽ bắt đầu xây dựng dự án Portfolio (Hồ sơ năng lực) chuẩn mực.

  • Tạo thư mục: Tạo thư mục mới My_Portfolio. Bên trong tạo 2 file: index.html (Trang chủ) và hobbies.html (Trang sở thích).
  • Khởi tạo: Gõ ! ở index.html để tạo bộ khung cơ bản.

Phần 2: Phân tích và code Header & Menu Navigation (35 Phút)

Khái niệm HTML Semantic (Ngữ nghĩa):

"Để Google dễ dàng lập chỉ mục (SEO) cho trang web của bạn, chúng ta cần dùng các thẻ có ý nghĩa rõ ràng thay vì nhét mọi thứ vào thẻ <div> chung chung: <header> (phần đầu trang), <nav> (thanh điều hướng), <main> (nội dung chính), <footer> (cuối trang)."

Thực hành: Gõ đoạn code sau vào trong thẻ <body> của file index.html:

<!-- Phần thanh menu điều hướng -->
<nav>
    <h2>OSCAR.DEV</h2> <!-- Thay bằng Tên viết tắt của bạn -->
    <ul>
        <li><a href="index.html">Trang Chủ</a></li>
        <li><a href="hobbies.html">Sở Thích Của Tôi</a></li>
        <li><a href="#contact">Liên Hệ</a></li>
    </ul>
</nav>

<!-- Phần Giới thiệu bản thân -->
<header>
    <h1>Hey, I'm <br> [Tên Bạn]</h1>
    <p>17 tuổi | Sinh viên tương lai | Đam mê Cầu Lông & IT</p>
    <img src="link_anh_cua_ban.jpg" alt="Hình chân dung" width="200">
</header>

💡 Giải thích chi tiết:

  • <ul> (Unordered List): Danh sách không có số thứ tự (dạng dấu chấm bi).
  • <li> (List Item): Từng mục bên trong danh sách. <ul> là thẻ cha, <li> là thẻ con.
  • <a> (Anchor): Thẻ quan trọng bậc nhất tạo nên khái niệm "Web". Dùng để tạo link bấm chuyển trang. Thuộc tính href="link_đích" chỉ đường dẫn đến nơi cần tới.

Buổi 3 MẶC ÁO CHO WEB VỚI CSS EXTERNAL & BOX MODEL (90 Phút)

Phần 1: Tách file CSS và Kết nối (20 Phút)

Giới thiệu: Nếu viết CSS thẳng vào file HTML như buổi 1, mã nguồn sẽ trở nên cực kỳ rối mắt khi dự án lớn lên. Việc này giống như việc bạn đi dây cáp (Cable Management) cho PC, mọi thứ cần được giấu gọn gàng. Do đó, chúng ta sẽ tách CSS ra một file hoàn toàn riêng biệt.

Thực hành:

  • Tạo một file mới tên là style.css nằm cùng thư mục với index.html.
  • Mở file index.html. Ở trong khu vực <head>, gõ chữ link rồi ấn nút Tab để VS Code tự động tạo dòng liên kết file.
<head>
    <meta charset="UTF-8">
    <title>My Portfolio</title>
    <link rel="stylesheet" href="style.css">
</head>

Phần 2: "Thiết quân luật" & Khái niệm Box Model (40 Phút)

1. Reset CSS cơ bản:

Mỗi trình duyệt web (Chrome, Safari, Edge) có một kiểu hiển thị lề mặc định khác nhau. Việc đầu tiên của mọi lập trình viên Front-end là phải 'Reset' lại thông số về 0 để làm chủ hoàn toàn giao diện.

/* Dấu sao (*) đại diện cho TẤT CẢ các thẻ trên trang web */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

2. Giải thích Box Model (RẤT QUAN TRỌNG):

"Trong thiết kế Web, mọi phần tử đều là những cái hộp hình chữ nhật. Hãy tưởng tượng bạn mua một đôi giày đựng trong hộp carton:
  • Content (Nội dung): Chính là đôi giày.
  • Padding (Lớp lót): Là mút xốp chống sốc nhét bên TRONG hộp để bảo vệ giày. Khoảng cách này nới rộng nội dung ra mép viền.
  • Border (Đường viền): Là lớp vỏ carton cứng của cái hộp.
  • Margin (Khoảng cách): Là khoảng cách từ hộp giày này đến hộp giày khác khi xếp trên kệ hàng. Lớp này nằm bên NGOÀI hộp.
Lệnh box-sizing: border-box; giúp máy tính hiểu rằng kích thước tổng thể mà ta khai báo đã bao gồm cả lớp mút xốp và vỏ hộp, giúp giao diện không bị phình to ngoài kiểm soát."

3. Bắt đầu Sơn phết nền và chữ:

Chuyển sang file style.css, chúng ta sẽ bắt đầu thiết lập màu sắc cho toàn bộ dự án.

body {
    background-color: #1a1a2e; /* Mã màu xanh đen nhám */
    color: #e94560; /* Chữ màu đỏ hồng Neon */
    font-family: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif;
}

h1 {
    font-size: 3rem; /* Chữ to gấp 3 lần bình thường */
    color: #0f3460;
}

p {
    font-size: 1.2rem;
    line-height: 1.5; /* Khoảng cách giữa các dòng chữ cho dễ đọc */
}

Phần 3: Làm đẹp từng phần tử (30 Phút)

Tiếp tục sử dụng Box Model để tạo ra khoảng không gian (thở) cho phần Header và Bo tròn ảnh đại diện.

header {
    text-align: center; /* Căn giữa toàn bộ chữ và hình trong header */
    padding: 50px 20px; /* Padding trên dưới 50px, trái phải 20px (Lớp mút xốp) */
    border-bottom: 2px solid #e94560; /* Vẽ một đường gạch dưới */
}

/* Trang trí ảnh đại diện */
header img {
    width: 200px;
    height: 200px;
    object-fit: cover; /* Tránh ảnh bị méo khi thu phóng */
    border-radius: 50%; /* Bo tròn 50% để thành hình tròn hoàn hảo */
    border: 4px solid #0f3460;
    margin-top: 20px; /* Đẩy ảnh cách xa đoạn văn bản phía trên (Margin - Khoảng cách kệ hàng) */
}

📝 Bài tập nhỏ: Bạn hãy thử tự thay đổi các thông số margin và padding rồi lưu file, quan sát sự thay đổi trên Live Server để cảm nhận rõ nhất nguyên lý hoạt động của Box Model.

Buổi 4 SẮP XẾP ĐỘI HÌNH VỚI FLEXBOX (90 Phút)

Phần 1: Khái niệm Flexbox (20 Phút)

Giới thiệu: Flexbox là "vũ khí tối thượng" trong CSS hiện đại dùng để bố trí giao diện. Hãy tưởng tượng Flexbox như một Huấn luyện viên thể thao (Thẻ Cha). HLV này có quyền ra lệnh cho các cầu thủ (Thẻ Con) phải đứng dàn hàng ngang, xếp hàng dọc, hoặc tản đều ra hai bên sân.

Phân tích bài toán: Thanh Menu hiện tại đang là một danh sách <ul> dọc đứng với các dấu chấm tròn, chiếm quá nhiều diện tích. Chúng ta sẽ dùng Flexbox để ép nó nằm ngang trên cùng: Logo nằm sát mép trái, các nút Menu nằm sát mép phải.

Phần 2: Áp dụng Flexbox cho Thanh Menu Navigation (40 Phút)

Thực hành: Mở file style.css và thêm các đoạn lệnh sau:

/* 1. Xử lý thanh 
 tổng (Thẻ Cha) */
nav {
    display: flex; /* Kích hoạt sức mạnh Flexbox */
    justify-content: space-between; /* Đẩy logo sang trái, menu sang phải */
    align-items: center; /* Căn giữa theo chiều dọc */
    background-color: #16213e; /* Nền tối hơn body một chút */
    padding: 15px 30px;
}

/* 2. Xử lý tên Logo */
nav h2 {
    color: #e94560;
    font-size: 1.5rem;
    cursor: pointer; /* Đưa chuột vào biến thành hình bàn tay */
}

/* 3. Xử lý danh sách 
     (Nó là thẻ con của 
    , nhưng lại là thẻ Cha của 
  • ) */
nav ul {
    display: flex; /* Tiếp tục dùng Flexbox để các nút menu nằm ngang */
    gap: 30px; /* Khoảng cách giữa các nút menu là 30px */
    list-style: none; /* Xóa dấu chấm tròn gạch đầu dòng */
}

/* 4. Xử lý các đường link  bên trong menu */
nav ul li a {
    color: #ffffff;
    text-decoration: none; /* Xóa đường gạch chân mặc định của link */
    font-weight: bold;
    transition: 0.3s; /* Hiệu ứng chuyển động mượt 0.3 giây */
}

/* Tạo hiệu ứng Hover (khi rê chuột vào phần tử) */
nav ul li a:hover {
    color: #e94560;
}

Phần 3: Áp dụng Flexbox cho Khu vực Sở thích (30 Phút)

Bây giờ chúng ta sẽ áp dụng Flexbox để dàn đội hình cho trang hobbies.html.

Bước 1 (Sửa HTML): Mở file hobbies.html, sử dụng một <div> chung bao bọc toàn bộ các sở thích để tạo thành Thẻ Cha.

<main>
    <h1 class="page-title">Sở Thích Của Tôi</h1>
    
    <div class="hobbies-container">
        <div class="hobby-card">
            <h2>1. Chơi Game</h2>
        </div>
        
        <div class="hobby-card">
            <h2>2. Cầu Lông & Thể Thao</h2>
            <p>Cao 1m80 là lợi thế lớn trên lưới...</p>
        </div>
    </div>
</main>

Bước 2 (Thêm CSS): Chuyển sang style.css để thiết lập Flexbox.

/* Thiết kế thẻ Cha */
.hobbies-container {
    display: flex;
    justify-content: center; /* Căn giữa đội hình */
    gap: 40px; /* Các hộp cách nhau 40px */
    padding: 50px;
    flex-wrap: wrap; /* TÍNH NĂNG QUAN TRỌNG: Tự động xuống dòng nếu màn hình không đủ chỗ */
}

/* Thiết kế thẻ Con (Từng hộp sở thích) */
.hobby-card {
    background-color: #16213e;
    padding: 20px;
    border-radius: 10px;
    width: 400px; /* Chiều rộng cố định */

Giới thiệu: HTML và CSS giúp web đẹp, nhưng nó giống như một bức tranh tĩnh. Muốn web có thể tính toán, phản hồi khi click chuột, ẩn hiện các thông báo... chúng ta cần đến JavaScript (JS) - hệ thống thần kinh của Website.

Thực hành thiết lập:

  • Tạo file main.js nằm cùng thư mục dự án.
  • Lưu ý: File JS chứa nhiều thao tác logic nặng, nên người ta thường nhúng nó ở tận cùng của file HTML (ngay trước thẻ đóng </body>). Điều này giúp trình duyệt ưu tiên load xong giao diện hình ảnh trước rồi mới load đến phần xử lý, tạo cảm giác mượt mà cho người dùng.
    <!-- Nằm ngay trước khi kết thúc body -->

Phần 2: Biến (Variable) & Hàm (Function) - Tư duy Game thủ (35 Phút)

Có 2 khái niệm cốt lõi nhất trong lập trình cần nắm rõ:

  • Biến (Variable): Giống như Balo chứa đồ (Inventory) trong Game. Bạn dùng biến để chứa và lưu trữ các giá trị (Tên người dùng, điểm số, thông tin...). Biến thường được khai báo bằng từ khóa let.
  • Hàm (Function): Giống như một chuỗi Combo chiêu thức. Thay vì lần nào cũng gõ lại 5 thao tác, bạn gom 5 thao tác đó vào một 'Hàm'. Khi cần chạy, chỉ cần gọi tên Hàm là chuỗi hành động được thực thi tự động.

Viết Hàm đầu tiên trong main.js:

// Tạo một hàm (combo) tên là chaoHoi
function chaoHoi() {
    // let là tạo cái balo chứa tên
    let tenNguoiChoi = "Gamer";
    
    // alert là lệnh bật hộp thoại thông báo trên màn hình
    alert("Xin chào " + tenNguoiChoi + "! Chào mừng bạn đến với Web.");
}

Gắn sự kiện bên file HTML: Hàm đã tạo xong nhưng nó cần một "công tắc" kích hoạt. Chúng ta sẽ tạo một nút bấm (Button) trong HTML và gắn sự kiện onclick vào nó.

<div style="text-align: center; margin: 30px;">
    <button onclick="chaoHoi()" style="padding: 10px 20px; font-size: 16px; cursor: pointer;">
        Bấm vào đây để test!
    </button>
</div>

Buổi 6 THAO TÁC DOM & HOÀN THIỆN THUẬT TOÁN BMI (90 Phút)

Phần 1: Kỹ năng "Bắn tỉa" - Lấy dữ liệu bằng ID (30 Phút)

Khái niệm DOM: Để Javascript có thể lấy được dữ liệu mà người dùng vừa gõ trên file HTML, nó cần dựa vào các thuộc tính id. ID giống như "biển số xe" duy nhất của mỗi thẻ. Lệnh quan trọng nhất để làm việc này là: document.getElementById('tên_id').

function tinhBMI() {
    // 1. Lấy giá trị (value) người dùng gõ vào ô chiều cao
    let theChieuCao = document.getElementById('chieuCao');
    let giaTriChieuCao = theChieuCao.value;
    
    // 2. Lấy giá trị (value) người dùng gõ vào ô cân nặng
    let theCanNang = document.getElementById('canNang');
    let giaTriCanNang = theCanNang.value;
    
    // In thử ra Console để kiểm tra xem đã lấy đúng chưa (Kỹ năng Debug)
    console.log("Chiều cao nhập vào:", giaTriChieuCao);
    console.log("Cân nặng nhập vào:", giaTriCanNang);
}

Kỹ năng gỡ lỗi (Debug): Hãy mở trình duyệt, ấn phím F12, chuyển sang tab Console. Nhập thử vài số và ấn nút Tính Toán trên giao diện. Bảng Console này chính là hậu trường dành riêng cho Lập trình viên để kiểm soát luồng dữ liệu.

Phần 2: Thuật toán và Toán học cơ bản (30 Phút)

Công thức tính BMI quốc tế là: Cân nặng chia cho bình phương chiều cao (tính bằng mét). Tuy nhiên giao diện web đang yêu cầu nhập chiều cao bằng cm, do đó bước đầu tiên là phải quy đổi đơn vị.

    // 3. Xử lý thuật toán (Viết tiếp vào bên TRONG hàm tinhBMI)
    let met = giaTriChieuCao / 100; // Đổi 180cm thành 1.8m
    let bmi = giaTriCanNang / (met * met); // Cân nặng chia cho (chiều cao x chiều cao)
    
    // Làm tròn kết quả chỉ lấy 1 chữ số thập phân cho gọn
    bmi = bmi.toFixed(1);
    console.log("Chỉ số BMI tính ra là:", bmi);

Phần 3: Lệnh Điều Kiện (If/Else) & Trả kết quả (30 Phút)

Giải thích If/Else: Tương tự như việc rẽ nhánh trong game cốt truyện. Máy tính sẽ đánh giá biến bmi vừa tính được nằm trong vùng nào để xuất ra lời khuyên tương ứng.

    // 4. Xét điều kiện để đưa ra lời khuyên
    let loiKhuyen = ""; // Tạo một balo rỗng để chứa câu thông báo

    if (bmi < 18.5) {
        loiKhuyen = "Hơi ốm rồi! Cần ăn thêm nhiều dinh dưỡng và tăng cường tập luyện.";
    } 
    else if (bmi >= 18.5 && bmi <= 24.9) { // Ký hiệu && nghĩa là 'VÀ'
        loiKhuyen = "Cơ thể chuẩn mực! Duy trì chế độ sinh hoạt này nhé.";
    } 
    else {
        loiKhuyen = "Bắt đầu thừa cân rồi. Cần bổ sung Cardio và chạy bộ giảm mỡ nhen.";
    }

    // 5. Thọc tay vào HTML, tìm thẻ 
 có ID là 'ketQua' đang để trống, và ép chữ vào (Thao tác DOM Manipulation)
    let dongKetQua = "BMI của bạn là: " + bmi + " - " + loiKhuyen;
    document.getElementById('ketQua').innerText = dongKetQua;
} // Đóng ngoặc nhọn của hàm tinhBMI

🎉 Chúc mừng bạn:

Đến bước này, khi nhập thông số (VD: 180, 75) và ấn TÍNH TOÁN, trang web đã có thể tự động trả về kết quả mượt mà không cần load lại trang. Đây chính xác là những gì mà kiến trúc cốt lõi của các Web-App hiện đại (như Facebook, Shopee...) đang thực hiện bên dưới: Lấy dữ liệu -> Xử lý (If/Else) -> Trả về màn hình.

Buổi 7 & 8 RESPONSIVE - ĐƯA WEB LÊN ĐIỆN THOẠI (180 Phút)

Phần 1: Bắt lỗi giao diện & Khái niệm Media Query (Buổi 7 - 45 Phút)

Đặt vấn đề: Giao diện web của chúng ta xem trên màn hình PC thì cực kỳ hợp lý. Tuy nhiên, nếu bạn ấn F12 và chuyển sang chế độ hiển thị Điện thoại di động (Mobile View), các thành phần có thể sẽ đè lên nhau hoặc bị tràn viền gây mất thẩm mỹ.

Giải pháp: Sử dụng Media Queries trong CSS. Bạn có thể hiểu đây là một "cảm biến" đo chiều rộng màn hình. Khi nó phát hiện chiều rộng thiết bị nhỏ hơn một mốc nhất định (thường là 768px), nó sẽ kích hoạt bộ giao diện CSS dự phòng.

Thực hành (Mở file style.css, bắt buộc viết ở DƯỚI CÙNG của file):

/* --- KHU VỰC DÀNH RIÊNG CHO ĐIỆN THOẠI --- */
/* Cảm biến kích hoạt khi chiều rộng màn hình <= 768px */
@media screen and (max-width: 768px) {
    /* 1. Xử lý thanh Menu */
    nav {
        flex-direction: column; /* Thay đổi đội hình Flexbox từ ngang thành dọc */
        padding: 10px;
    }
    nav ul {
        margin-top: 15px;
        flex-direction: column; /* Các nút bấm menu xếp chồng xuống dưới */
        gap: 10px;
        text-align: center;
    }
    
    /* 2. Xử lý chữ Header */
    h1 {
        font-size: 2rem; /* Thu nhỏ tiêu đề lớn để không bị tràn dòng */
    }
}

Phần 2: Tối ưu Công cụ BMI & Sở thích trên Điện thoại (135 Phút)

    /* Viết tiếp đoạn mã này NẰM BÊN TRONG cặp ngoặc nhọn của @media phía trên */

} /* KẾT THÚC KHỐI LỆNH @MEDIA */

/* Thiết lập giao diện màu sáng */

function doiGiaoDien() {
